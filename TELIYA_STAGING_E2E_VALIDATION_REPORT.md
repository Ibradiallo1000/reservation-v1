# Validation E2E staging

Statut : non exécutée. La configuration staging existe (`teliya-staging`, mode Vite staging), mais l’environnement Codex ne peut pas établir une connexion Firebase CLI fiable à cause du certificat TLS local. Aucune donnée réelle n’a donc été inventée.

Restent à exécuter avec comptes et données existants : Mali (Bamako–Ségou), Côte d’Ivoire (Abidjan–Bouaké si disponible), Marketplace → résultats → compagnie → départ → réservation, accès direct `/:slug`, sous-domaine, domaine personnalisé, QR/lien partagé, puis refresh aux étapes résultats, réservation, paiement et confirmation. Les états sans pays, inactive, sans trajet, vide, indisponible, paiement en attente et billet doivent être consignés.
